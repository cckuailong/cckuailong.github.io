Ubuntu HBase安装配置Snappy压缩模式

2020-03-17

一、前言

HBase存储大量数据时，压缩可以充分利用存储空间，本文使用Snappy，在Ubuntu环境下
对HBase进行配置。

二、环境

  • Ubuntu 16.04
  • Hadoop-2.7.7
  • ZooKeeper-3.5.6
  • HBase-1.4.12
  • Snappy-1.1.8

三、安装snappy

  • 下载源码
https://src.fedoraproject.org/repo/pkgs/snappy/
  • 解压
tar zxvf snappy-1.1.8.tar.gz -C /root/
  • 编译
cd /root/snappy-1.1.8/
mkdir build && cd build/
cmake ../ && make && make install

此时在 /usr/lib/x86_64-linux-gnu/ 中会生成libsnappy开头的几个文件，将他们复制

cp /usr/lib/x86_64-linux-gnu/libsnappy* /usr/local/lib/

四、安装hadoop-snappy

  • 下载源码
cd ~
git clone https://github.com/electrum/hadoop-snappy --depth=1
  • 安装依赖
apt install autoconf automake libtool maven
  • 编译
cd hadoop-snappy
mvn package

成功后，会生成一个target文件夹

  • 验证
hadoop checknative

可以看到snappy为true

五、配置HBase

  • 获得编译好的lib
cd /root/src/hadoop-snappy/target
tar zxvf hadoop-snappy-0.0.1-SNAPSHOT.tar.gz
cd hadoop-snappy-0.0.1-SNAPSHOT/lib

这里面的lib就是编译好的snappy的lib

  • 拷贝lib

1) 拷贝snappy native lib

cp hadoop-snappy-0.0.1-SNAPSHOT.jar /root/hbase/lib/
mkdir /root/hbase/lib/native
cp -r native/Linux-amd64-64/ /root/hbase/lib/native/

2) 拷贝hadoop native lib

cp /root/hadoop/lib/native/* /root/hbase/lib/native/Linux-amd64-64/
  • 修改环境变量

1) vim /etc/profile，添加

export LD_LIBRARY_PATH=/root/hadoop/lib/native
export PATH=$PATH:$LD_LIBRARY_PATH
export HBASE_LIBRARY_PATH=/root/hbase/lib/native/Linux-amd64-64
export PATH=$PATH:$HBASE_LIBRARY_PATH

2) source /etc/profile

  • 修改 HBase 配置文件

vim /root/hbase/conf/hbase-site.xml，添加

<property>
    <name>io.compression.codecs</name>
    <value>snappy</value>
</property>

六、分发给slave

  • 分发native lib
scp /usr/local/lib/libsnappy* root@slave*:/usr/local/lib/
scp /root/hbase/lib/hadoop-snappy-0.0.1-SNAPSHOT.jar  root@slave*:/root/hbase/lib/
scp -r /root/hbase/lib/native/  root@slave*:/root/hbase/lib/
  • 配置环境变量

同master

  • HBase 配置文件
scp /root/hbase/conf/hbase-site.xml root@slave*:/root/hbase/conf/

七、重启HBase

/root/hbase/bin/stop-hbase.sh
/root/hbase/bin/start-hbase.sh

八、验证

  • snappy压缩HDFS上的文件
/root/hbase/bin/hbase org.apache.hadoop.hbase.util.CompressionTest hdfs://master:9000/tmp_rib/rrc10.txt snappy

末尾出现 SUCCESS，即为snappy安装成功

  • HBase中创建带压缩属性的表
create 'bgp:history', { NAME => 'record', COMPRESSION => 'SNAPPY' }

创建成功，HBase部署snappy成功
