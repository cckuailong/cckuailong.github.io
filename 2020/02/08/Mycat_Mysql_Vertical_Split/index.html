Mycat基于Mysql的垂直切分

2020-02-08

一、前言

Mycat可以对数据库进行分库分表管理，本文针对Mycat的垂直切分功能进行实验，
实现不同的表，分布在不同的机器的数据库中。本文使用Mycat对Mysql进行操作。

二、环境

  • Mycat 1.6.7.5
  • Mysql 5.7

  • openjdk-8

  • mycat master: 192.168.118.128(domain: server)

  • mycat slave1: 192.168.118.129(domain: slave1)

数据库名

  • master, slave1: tp_bgp_data

表名

  • master: link, origin
  • slave1: monitor

三、安装配置

安装 mysql

设置为允许远程连接

下载mycat

master上下载并解压Mycat

wget http://dl.mycat.io/1.6.7.5/Mycat-server-1.6.7.5-test-20200109231555-linux.tar.gz
tar zxvf Mycat-server-1.6.7.5-test-20200109231555-linux.tar.gz

配置Mycat

配置文件在 ~/mycat/conf

  • schema.xml
<?xml version="1.0"?>
<!DOCTYPE mycat:schema SYSTEM "schema.dtd">
<mycat:schema xmlns:mycat="http://io.mycat/">

    <schema name="TESTDB" checkSQLschema="true" sqlMaxLimit="100" randomDataNode="dn1">
        <table name="link" dataNode="dn1"/>
        <table name="origin" dataNode="dn1"/>
        <table name="monitor" dataNode="dn2"/>
    </schema>

    <dataNode name="dn1" dataHost="master" database="tp_bgp_data" />
    <dataNode name="dn2" dataHost="slave1" database="tp_bgp_data" />

    <dataHost name="master" maxCon="1000" minCon="10" balance="0"
              writeType="0" dbType="mysql" dbDriver="native" switchType="1"  slaveThreshold="100">
        <heartbeat>select user()</heartbeat>

        <writeHost host="hostM1" url="localhost:3306" user="root"
                   password="1q2w3e4r">
        </writeHost>
    </dataHost>

    <dataHost name="slave1" maxCon="1000" minCon="10" balance="0"
              writeType="0" dbType="mysql" dbDriver="native" switchType="1"  slaveThreshold="100">
        <heartbeat>select user()</heartbeat>

        <writeHost host="hostM1" url="slave1:3306" user="root"
                   password="1q2w3e4r">
        </writeHost>
    </dataHost>

</mycat:schema>

四、实验

启动Mycat

master上运行

~/mycat/bin/mycat start

检查是否启动成功

cat ~/mycat/logs/wrapper.log

若显示successful，则启动成功，否则，查看其中的报错，可能是配置错误，可能是环境错误（例：java环境未安装）

操作Mycat

  • 登陆mycat

登录到Mycat的逻辑数据库TESTDB（server.xml中可修改）

mysql -uroot -p1q2w3e4r -hmaster -P8066 -DTESTDB
  • 查看表
show tables;

发现master和slave1主机的tp_bgp_data数据库中的表都在

  • 写数据
insert into monitor values(1, "aabbcc");
  • 读数据
select * from monitor;

结果

mysql> select * from monitor;
+------+---------+
| id   | msg_md5 |
+------+---------+
|    1 | aabbcc  |
+------+---------+

五、总结

以上就是Mycat的垂直切分示例，使用Mycat中间件可以使我们操作分布式数据库，就像操作单一数据库一样，
对于单数据库切换到分布式数据库有着十分重要的意义，并且原项目不需进行代码改动。
