Apache服务器安全--Ubuntu上的Apache安全配置（二）

2017-06-23

一、前言：

上一章，我们列举了一系列Ubuntu端Apache配置的安全防御措施，这一章我们将继续来关注
Apache安全配置策略。这一章主要介绍一些先进的小技术来使你的服务器变得更加安全。

二、安全问题及配置策略：

1、防御点击劫持攻击

点击劫持是著名的web服务器攻击，也被称作UI-覆盖攻击，他通过覆盖不可见的框架（iframe）
误导受害者点击，虽然点击的是看到的网页，但其实点击的是hacker精心构造的另一个透明的
网页，主要利用了HTML中iframe的透明属性。

这里配置需要用到X-Frame-Options,有三个选项：
DENY：浏览器拒绝当前页面加载任何frame页面
SAMEORIGIN：frame页面的地址只能为同源域名下的页面（协议，ip，端口号相同）
ALLOW-FROM：origin为允许的frame加载的页面地址

安全配置策略：
（1） 修改Apache配置文件

sudo vim /etc/apache2/apache2.conf

（2） 在/var/www/html中增添或修改

Header always append X-Frame-Options SAMEORIGIN

（3）重启Apache

sudo /etc/init.d/apache2 restart

（4）注意，重启时可能会出现下面的错误

Invalid command ‘Header’, perhaps misspelled or defined by a module not included in the server configuration

由于没有安装headers模块，需要先行安装

sudo a2enmod headers

安装完毕后，重启成功

配置完成效果，访问应答包中出现了X-Frame-Options：SAMEORIGIN

2、禁用Etag

Etag是服务器和浏览器的一个功能，用来判断浏览器缓存里的元素是否和原来服务器上的
一致。Apache默认的Etag格式为inode-size-timestamp，暴露了一些敏感信息，可以修改Etag的
格式来保护敏感信息，但最好的方法是将Etag关闭，因为Etag还会引发一些其他的问题。

安全配置策略：

（1）修改apache2.conf

sudo vim /etc/apache2/apache2.conf

（2）对Directory /var/www/html 增添或修改
关闭

FileETag None

修改Etag格式

FileETag MTime Size  //格式可以自己根据需要修改

（3）重启Apache

sudo /etc/init.d/apache2 restart

配置完成后，应答包中将不再有Etag标签

3、禁止过时的协议

过时的HTTP协议（HTTP 1.0）在session劫持和点击劫持等方面存在脆弱性，推荐将其禁用，你可
以使用只允许HTTP 1.1协议的方式来禁用HTTP 1.0

安全配置策略：

（1）修改 apache2.conf

sudo vim /etc/apache2/apache2.conf

（2）对Directory /var/www/html 增添或修改：

RewriteEngine On
ReWriteCond %{THE_REQUEST} !HTTP/1\.1$
RewriteRule .* - [F]

（3）重启Apache

sudo /etc/init.d/apache2 restart

4、禁用不常用的HTTP请求方法

HTTP 1.1支持很多的请求方法，如”OPTIONS,GET,POST,HEAD,PUT,DELETE,TRACE,CONNECT”,有些方法
会使服务器收到安全威胁，如PUT方法会在服务器中写入文件，所以推荐不使用的方法就禁
用掉。

安全配置策略：

（1）修改 apache2.conf

sudo vim /etc/apache2/apache2.conf

（2）对Directory /var/www/html 增添或修改

deny from all

（3）重启Apache

sudo /etc/init.d/apache2 restart

5、防御XSS攻击

XSS攻击是最常见的应用层脆弱性攻击之一，攻击者通过用户的web浏览器在目标web服务器上
执行恶意代码，推荐在web服务器上进行XSS防护。

安全配置策略：

（1）修改apache2配置文件

sudo vim /etc/apache2/apache2.conf

（2）对Directory /var/www/html 增添或修改

Header set X-XSS-Protection "1; mode=block"

（3）重启Apache

sudo /etc/init.d/apache2 restart

配置完成效果，访问应答包中出现了X-X-XSS-Protection

6、使用HTTPOnly标志保护Cookies

网站的XSS漏洞会使用户被攻击者进行cookie的盗取，利用的是脚本获取cookie后发送到攻击
者的网站，而HTTPOnly标志会禁止脚本获取cookie，从而防止了盗取用户cookie。

安全配置策略：

（1）修改apache2配置文件

sudo vim /etc/apache2/apache2.conf

（2）对Directory /var/www/html 增添或修改

Header edit Set-Cookie ^(.*)$ $1;HttpOnly;Secure

（3）重启Apache

sudo /etc/init.d/apache2 restart
