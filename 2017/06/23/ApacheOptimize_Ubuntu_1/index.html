Apache服务器安全--Ubuntu上的Apache性能优化（一）

2017-06-23

一、前言：

Apache是一个开源的web服务器，在世界上拥有70%的份额。我们可以通过增加额外的硬件，如
RAM，CPU等来提高服务器的性能，同样我们也可以通过定制一些Apache配置来提高我们服务器
的性能。

二、优化配置策略：

1、MaxKeepAliveRequests

MaxKeepAliveRequests限制了一次连接所允许的请求数量，它被用来控制持久的连接。在Ubuntu
中，默认值为100。你可以根据需要修改这个值，推荐修改为50~75。

优化配置策略：
（1） 修改Apache主配置文件

sudo vim /etc/apache2/apache2.conf

（2） 增添或修改

MaxKeepAliveRequests 60

（3）重启Apache

sudo /etc/init.d/apache2 restart

2、KeepAliveTimeout

KeepAliveTimeout定义了服务器等待从连接的客户端发出的请求的时间。KeepAliveTimeout的值
太大可能会导致性能的降低。Ubuntu中默认值为15，推荐之在1~5之间。

优化配置策略：
（1）修改apache2.conf

sudo vim /etc/apache2/apache2.conf

（2）增添或修改

KeepAliveTimeout 3

（3）重启Apache

sudo /etc/init.d/apache2 restart

3、MaxClients

MaxClients限制web服务器同时提供服务的客户端数量，超过这个数量的客户端连接请求将被
放置在排队队列中，一旦有客户端结束链接，便从排队队列中加入到服务队列。在Ubuntu中，
默认的MaxClient值为250.推荐值为150

优化配置策略：

（1）修改mpm_prefork.conf文件

sudo vim /etc/apache2/mods-available/mpm_prefork.conf

（2）增添或修改

MaxClients 150

（3）重启Apache

sudo /etc/init.d/apache2 restart

4、MaxConnectionsPerChild

MaxConnectionsPerChild用来回收进程。当限制被设置为0时，每个进程允许无限制的请求。
MaxConnectionsPerChild限制一个子进程能处理的请求的数量，在请求达到指定限制时，子
进程就会消亡。在Ubuntu中，默认值为100。

推荐的设置：
● 虚拟服务器 300
● 1-4GB RAM的服务器 500
● 4+GB RAM的服务器 1000

优化配置策略：

（1）修改mpm_prefork.conf

sudo vim /etc/apache2/mods-available/mpm_prefork.conf

（2）添加或修改

MaxConnectionsPerChild 300

（3）重启Apache

sudo /etc/init.d/apache2 restart

KeepAlive

Ubuntu默认将KeepAlive设置为On。当Apache服务器短时间内收到成百上千的IP请求，KeepAlive值
应该被设置为Off。推荐将此值设置为Off来提高连接吞吐量。

优化配置策略：

（1）修改apache2配置文件

sudo vim /etc/apache2/apache2.conf

（2）增添或修改：

KeepAlive Off

（3）重启Apache

sudo /etc/init.d/apache2 restart

6、MinSpareServers and MaxSpareServers

这两个参数设置了空闲服务器子进程的最大和最小数量。他控制Apache保持空闲状态的进程
的数量，直到活得更多的请求来重新激活他们。每一个子进程都消耗一定的资源，所以如果
你把MaxSpareServer设置的太大，将会导致资源紧张问题。

MinSpareServers的推荐设置为
● 虚拟服务器 5
● 1-2GB RAM的服务器 10
● 2-4GB RAM的服务器 20
● 4+GB RAM的服务器 25

MaxSpareServers推荐设置为MinSpareServer的两倍

优化配置策略：

（1）修改mpm_prefork.conf配置文件

sudo vim /etc/apache2/mods-available/mpm_prefork.conf

（2）增添或修改：

MinSpareServers 5
MaxSpareServers 10

（3）重启Apache

sudo /etc/init.d/apache2 restart
