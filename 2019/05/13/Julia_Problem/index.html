Julia问题及解决方案整理

2019-05-13

一、前言：

Julia目前最新版本为1.1，由于是新语言，还是有一些问题，这里做一个整理，并
不断更新中。

二、Version 1.1：

Notepad++配置Julia

参考我的另一篇文章，Notepad++配置python
只需要修改python安装目录为julia安装目录即可。

P.S. Notepad++配置Julia高亮插件

更换Julia环境目录

Julia默认环境目录为

~/.julia

安装的各种包都下载在里面，会导致系统盘负载较高，需要更换其位置。
增加系统环境变量

JULIA_DEPOT_PATH=你想要设置的环境目录
输入 ]up 进行更新（必要）

再次运行julia发现工作环境目录改变

更换国内源

(v1.1) pkg> add PkgMirrors
julia> import PkgMirrors
julia> PkgMirrors.availables()
julia> PkgMirrors.setmirror("ZJU")
julia> import PkgMirrors
(v1.0) pkg> update

update时，可能会出Warning

Warning: Some registries failed to update:

这时，进入到你的环境目录（不改的话，默认为 ~/.julia/）

cd registries/General
git stash
git pull
git stash drop

再update，Warning消失

Julia Build时需要翻墙

解决办法：

  1. 购买VPN（ss等，配置成全局代理，P.S. PAC不行）
  2. 根据错误提示下载相应的包，放到提示的目录中（比较麻烦，但是可行）

Atom package目录更改

Atom默认package目录为

~/.atom
ATOM_HOME=你想要设置的package目录

发现Atom安装package目录已更改成功

Atom 更改软件源

apm config set registry http://registry.npm.taobao.org

创建第三方package

import Pkg
Pkg.add("PkgTemplates")

创建package脚本

using PkgTemplates

t = Template(;
           user="Your Github Username",
           authors=["The Author"],
           dir="Package生成到哪个目录",
           julia_version=v"1.1",   # 最低julia编译版本
           plugins=[
               TravisCI(),         # build pass
               Coveralls(),        # coverage
           ],
       )
generate(t, "Package Name")
run(`git -C $(joinpath(t.dir, "Package Name")) ls-files`);

向Project.toml中添加依赖包

cd 到package目录
] 进入Pkg模式
activate .
add 依赖包
