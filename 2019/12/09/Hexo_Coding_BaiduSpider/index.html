Hexo进行Coding Page分流，实现百度爬虫收录

2019-12-09

一、前言

Hexo + GitPage服务搭建个人博客已经非常流行，搭建方法详见 Hexo Your Blog,
但是由于github限制了百度的爬虫抓取，所以git page搭建的博客不会被百度爬取收录，
为了使百度收录我们的blog，分享更多的知识给大家，特查询总结了Coding Page进行分流
的方法来实现百度爬虫收录。

注意

全文 以我的项目为例

  • Github仓库(https://github.com/cckuailong.github.io)，
  • Coding仓库(https://git.dev.tencent.com/cckuailong/cckuailong.coding.me)
  • 域名(www.lovebear.top)

二、Coding Page注册

  • 访问 https://coding.net ，点击右上角个人版登陆，然后进行注册，登陆等常规操作。

  • 创建新项目，这里取名为 cckuailong.coding.me

  • 点击右上角的头像，进入个人设置，点击右方SSH公钥，参考github上传ssh公钥的方法，
    见 Hexo Your Blog，将openssh生成的公钥上传

  • P.S. 上传了ssh公钥还是需要输入密码，可能因为公钥太老了，重新生成公私钥对，并上传公钥即可

三、将Git Page的代码转移到Coding Page

  • 将GitPage仓库clone到本地
git clone https://github.com/cckuailong/cckuailong.github.io
  • 将clone下来的仓库代码上传到coding
cd cckuailong.github.io
git push https://git.dev.tencent.com/cckuailong/cckuailong.coding.me.git --all

若 ssh公钥 上传成功，则不需要输入密码，否则根据提示输入coding注册的用户名和密码

四、开启Page服务

  • 进入新创建的项目，点击 代码–Page服务

  • 开启Page服务，此时访问 https://cckuailong.coding.me 即可访问Page页面

  • 点击右上角 设置

  • 输入要绑定的域名

www.lovebear.top
lovebear.top

五、设置DNSpod

思想是国内解析域名到Coding Page, 国外解析到Git Page

直接给出我的配置，供参考

主机记录 记录类型 线路类型 记录值
@ CNAME 境外 cckuailong.github.io.
@ CNAME 境内 cckuailong.coding.me.
www CNMAE 默认 lovebear.top.
@ NS 默认 f1g1ns1.dnspod.net.
@ NS 默认 f1g1ns2.dnspod.net.

六、设置hexo提交新内容时，同时提交到GitPage和CodingPage

  • Hexo中的 _config.yml文件的deploy，修改为
deploy:
- type: git
  repo:
        github: git@github.com:cckuailong/cckuailong.github.io.git
        coding: git@git.coding.net:cckuailong/cckuailong.coding.me.git
  branch: master
  • 这时，就实现了博客分流，并且因为国内ip会解析到 coding.me, 所以，也实现了百度抓取。
    试一试给百度提交链接，我们的博客就能被百度收录了。
